RogueOne: MITRE ATT&CK Map

DATE: 2026-04-19 | TAGS: mitre att&ck c2-analysis threat-hunting

The RogueOne implant utilizes highly sophisticated techniques spanning multiple MITRE ATT&CK tactics to establish persistence, elevate privileges, and evade local endpoint defenses. The following mapping details the exact implementation of these techniques within the implant's source code, alongside actionable mitigation strategies for defenders.

::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::

TA0005: Defense Evasion & TA0002: Execution

T1055.012 - Process Injection: Process Hollowing

Implementation: The implant relies heavily on Process Hollowing to execute arbitrary shellcode in memory without touching the disk. It spawns a benign process (notepad.exe) in a suspended state using the CREATE_SUSPENDED flag. It then uses NTAPIs like NtQueryInformationProcess to locate the Process Environment Block (PEB), calculates the entry point offset, overwrites the target's legitimate executable code with malicious shellcode using WriteProcessMemory, and executes it via ResumeThread.

Additionally (T1134.004 - Parent PID Spoofing): To mask the true origin of the hollowed process, the implant dynamically resolves the PID of explorer.exe and uses InitializeProcThreadAttributeList alongside UpdateProcThreadAttribute to assign explorer.exe as the hollowed process's parent.

Mitigation & Remediation

Implement EDR solutions capable of monitoring API calls related to thread context manipulation (GetThreadContext / SetThreadContext) and memory modifications in newly spawned suspended processes. Enable Windows Defender Application Control (WDAC) and monitor for anomalous Parent-Child process relationships (e.g., an unexpected process suddenly spawning notepad.exe with explorer.exe as the spoofed parent).

T1027 - Obfuscated Files or Information

Implementation: The developer attempted to hide critical strings (such as registry keys and executable paths) from static analysis tools. In uac.c, the target registry path Software\Classes\Launcher.SystemSettings\Shell\Open\Command is Base64 encoded inside the binary as UwBvAGYAdAB3AGE.... The implant uses the native Cryptography API (CryptStringToBinaryW) inside enc.c to dynamically decode these strings at runtime.

Mitigation & Remediation

Relying purely on static string signatures is ineffective against this. Utilize dynamic analysis (sandboxing) or memory scanning (like YARA rules scanning active process memory) to capture the decoded strings after the implant decrypts them in RAM.

T1059.003 - Command and Scripting Interpreter: Windows Command Shell

Implementation: The implant implements a remote shell functionality via the runcmd feature. It calls CreateProcessW to spawn cmd.exe /c [command] and routes the STDOUT and STDERR streams through anonymous pipes (CreatePipe) back to the socket.

Mitigation & Remediation

Enable Command Line Auditing (Event ID 4688) via Group Policy to log the exact arguments passed to cmd.exe. Monitor for unusual processes spawning cmd.exe with piped outputs.

TA0004: Privilege Escalation

T1548.002 - Bypass User Account Control (UAC)

Implementation: If the implant detects it is running in a medium-integrity context, it attempts a fileless UAC bypass. It achieves this by modifying the current user's registry to hijack the execution flow of an auto-elevating Windows binary. Specifically, it creates the key HKCU\Software\Classes\Launcher.SystemSettings\Shell\Open\Command, points the default value to its own executable path, and creates an empty DelegateExecute value. Finally, it triggers the payload by calling ShellExecuteW with the runas verb on slui.exe, forcing the OS to launch the implant with High Integrity.

Mitigation & Remediation

Configure UAC settings to "Always Notify" to prevent auto-elevation of binaries like slui.exe. Monitor registry modifications in the HKCU\Software\Classes\ hive, particularly looking for unusual Shell\Open\Command or DelegateExecute key creations.

T1134.001 - Access Token Manipulation: Token Impersonation/Theft

Implementation: Once running as an Administrator, the implant attempts to reach `NT AUTHORITY\SYSTEM`. It iterates through running processes to find winlogon.exe. Using OpenProcessToken and DuplicateTokenEx (requesting SecurityImpersonation and TokenPrimary), it steals the SYSTEM token. It then utilizes CreateProcessWithTokenW to respawn itself with ultimate system privileges.

Mitigation & Remediation

Enforce the principle of least privilege. Monitor for unusual processes acquiring the SeDebugPrivilege, which is required to open the token of winlogon.exe. Advanced EDRs should flag unauthorized binaries calling DuplicateTokenEx on highly privileged SYSTEM processes.

T1068 - Exploitation for Privilege Escalation (BYOVD)

Implementation: The implant utilizes a "Bring Your Own Vulnerable Driver" (BYOVD) technique to achieve Ring-0 (Kernel) code execution. It drops a payload to %TEMP%\MicrosoftUpdate11.01.sys and registers it as a kernel service. The code explicitly references CVE-2019-6494 and CVE-2025-26125. Once loaded, it issues DeviceIoControl commands to \\.\IMFForceDelete123 to forcefully delete files (likely targeting locked EDR files) from the kernel level.

Mitigation & Remediation

Enable Microsoft's Vulnerable Driver Blocklist via Windows Defender Application Control (WDAC). Enable Hypervisor-Protected Code Integrity (HVCI) to prevent unauthorized or known-vulnerable drivers from loading into the kernel space, regardless of Administrator privileges.

TA0003: Persistence

T1543.003 - Create or Modify System Process: Windows Service

Implementation: To survive reboots, the implant can install itself as a native Windows service. It uses the OpenSCManagerW and CreateServiceW APIs to create a service named WinTelemetry. It configures the service with the SERVICE_AUTO_START flag to ensure it executes automatically upon system boot, running as the LocalSystem account. It also sets a deceptive display name ("Windows Telemetry Agent") to blend in with legitimate OS functions.

Mitigation & Remediation

Monitor Windows Event Logs (System Event ID 7045 - A new service was installed in the system). Audit newly created services that point to binaries residing in unusual directories (like user profiles or %TEMP%). Restrict non-administrative users' ability to interact with the Service Control Manager.

TA0011: Command and Control

T1573.001 - Encrypted Channel: Symmetric Cryptography

Implementation: The implant completely avoids standard TLS/SSL to evade standard network decryptors. It establishes an AES-256-CBC encrypted tunnel over a raw TCP socket. The symmetric key is dynamically generated per-session using an Elliptic Curve Diffie-Hellman (ECDH P-256) key exchange over the wire.

Mitigation & Remediation

Because the implant uses custom crypto, traditional SSL inspection will fail. Rely on Network Traffic Analysis (NTA) to detect anomalous TCP connections to unknown external IPs, specifically looking for the 65-byte uncompressed public key signature (starting with 0x04) transmitted in plaintext during the initial ECDH handshake.
