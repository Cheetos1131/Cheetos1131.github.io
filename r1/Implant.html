Source Code Analysis: RogueOne Implant

DATE: 2026-04-19 | TAGS: c2 malware-analysis windows-internals

The provided source code constitutes an advanced Windows C2 implant agent dubbed RogueOne. Designed to operate stealthily on a compromised host, the agent communicates over an AES-256 encrypted TCP socket using a custom Type-Length-Value (TLV) protocol. It possesses extensive capabilities including Bring-Your-Own-Vulnerable-Driver (BYOVD) exploitation, process hollowing, token theft, and UAC bypasses.

1. General Program Flow

The lifecycle of the implant is driven by main.c and interactiveMain(). It is designed to act both as a standalone executable and as an installable Windows Service.

Execution Chain

  1. Service vs. Console: main() immediately attempts to register as a Windows service using StartServiceCtrlDispatcherW. If it fails (indicating it was executed interactively by a user), it falls back to InteractiveMain().
  2. Network Initialization: NetworkInit() is called to resolve the hardcoded C2 host/port and establish a TCP socket.
  3. Cryptographic Handshake: Over the plaintext socket, CryptoHandshake() executes an Elliptic Curve Diffie-Hellman (ECDH P-256) key exchange to derive a mutual 32-byte shared secret, subsequently hashed via SHA-256 to create the AES-CBC session key.
  4. Privilege Escalation check: If the implant detects it is running as an Administrator, it forcefully enables critical tokens: SeDebugPrivilege, SeTakeOwnershipPrivilege, SeLoadDriverPrivilege, and SeImpersonatePrivilege.
  5. C2 Loop: The agent enters an infinite while(RecvMessage(...)) loop. It waits for TLV packets, decrypts them, parses the CMD_ID, and routes the payload to the appropriate handler in the G_CommandTable via CommandDispatcher().

2. Detailed Module Breakdown

A. Protocol & Dispatch (protocol.c & command.c)

The network protocol is purely custom. Every packet sent to and from the C2 utilizes a strict TLV (Type-Length-Value) architecture defined by the TLV_MESSAGE struct.

  • Header: 8 bytes total (4 bytes for CMD_ID, 4 bytes for payload Length).
  • Chunking: Data larger than 1024 bytes (CHUNK_SIZE) is fragmented. An empty EOF packet (Length = 0) signals the end of a chunk stream.
  • Dispatch: command.c maps CMD_ID integers (e.g., CMD_PS, CMD_INJECT) to specific C functions using the G_CommandTable.

B. Cryptography (crypto.c & network.c)

The author elected to use native Windows CNG (Cryptography API: Next Generation) rather than embedding OpenSSL, significantly reducing the binary footprint and evading basic static signatures.

// Wire format of encrypted frames:
[4-byte LE Plaintext Length] + [16-byte IV] + [AES-256-CBC Ciphertext]

After the initial ECDH handshake, all traffic passes through NetworkSendAll and NetworkRecvAll, which transparently wrap the underlying socket calls with PKCS#7 padded AES encryption.

C. Evasion & Injection (hollower.c)

Process injection is accomplished using a technique called Parent PID Spoofing combined with Process Hollowing.

  1. Resolves the PID of explorer.exe using CreateToolhelp32Snapshot.
  2. Uses InitializeProcThreadAttributeList and UpdateProcThreadAttribute to assign explorer.exe as the parent of the new process.
  3. Spawns notepad.exe in a suspended state (CREATE_SUSPENDED).
  4. Traverses the PEB (Process Environment Block) to find the Image Base Address, calculates the Entry Point offset, writes malicious shellcode to the Entry Point via WriteProcessMemory, and calls ResumeThread.

D. Privilege Escalation (uac.c & token.c)

If the user is standard but UAC is enabled, the implant utilizes a registry-based UAC bypass targeting slui.exe.

  • Registry Bypass: Writes the path of the implant to HKCU\Software\Classes\Launcher.SystemSettings\Shell\Open\Command and creates an empty DelegateExecute key. Calling slui.exe with the runas verb automatically triggers the high-integrity payload execution without prompting the user.
  • Token Duplication: If already elevated, SpawnImplantWithToken locates winlogon.exe (which runs as NT AUTHORITY\SYSTEM), opens its process token, duplicates it via DuplicateTokenEx, and restarts the implant via CreateProcessWithTokenW to achieve SYSTEM level privileges.

E. Ring 0 / Kernel Mode (driver.c)

RogueOne implements a BYOVD (Bring Your Own Vulnerable Driver) attack to blind EDRs or forcibly delete locked files.

  • Drops a known vulnerable driver (referenced as CVE-2019-6494 / CVE-2025-26125) to %TEMP%\MicrosoftUpdate11.01.sys.
  • Installs the driver as a Kernel Service via the Windows Service Control Manager (SCM).
  • Issues raw Device IO Control (IOCTL) codes—specifically 0x8016E000—to the driver via \\.\IMFForceDelete123 to execute forced file deletion from ring 0.
