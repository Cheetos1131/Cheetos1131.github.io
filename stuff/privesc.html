Privilege Escalation Attack Flows

// educational reference — defensive security

UAC Bypass via Auto-Elevation Abuse PRIVILEGE ESCALATION
1
Check Elevation Status
Query process token to determine if already running as admin
OpenProcessToken → GetTokenInformation(TokenElevation)
2
Hijack Registry Key
Write to HKCU\Software\Classes\ to redirect a trusted binary's command lookup
HKCU\...\Shell\Open\Command → (Default) = "C:\malware.exe"
3
Set DelegateExecute = ""
Empty DelegateExecute forces COM to use the Shell\Open\Command path instead of the real handler
4
Trigger Auto-Elevating Binary
Launch a Microsoft-signed binary (e.g. slui.exe, fodhelper.exe) that auto-elevates without UAC prompt
ShellExecuteW("slui.exe") → auto-elevates → reads hijacked registry
5
Payload Executes as Admin
The trusted binary follows the hijacked registry path and launches the attacker's executable with full admin privileges — no UAC dialog shown
⚠ Result: Attacker code runs with admin privileges, UAC completely bypassed
BYOVD — Bring Your Own Vulnerable Driver KERNEL EXPLOITATION
1
Obtain Admin Privileges
Attacker already has admin access (e.g. via UAC bypass, credential theft, or social engineering)
2
Drop Legitimate Signed Driver
Install a known-vulnerable but validly signed kernel driver onto the system. It passes signature verification because it has a real certificate.
sc create vuln_driver binPath= "C:\path\to\driver.sys" type= kernel
3
Load Driver into Kernel
Start the driver service. Windows allows it because the driver is properly signed by a trusted vendor.
sc start vuln_driver → loaded into Ring 0 (kernel space)
4
Exploit Driver Vulnerability
Send crafted IOCTL calls to the driver to trigger a vulnerability (arbitrary read/write, code execution in kernel)
DeviceIoControl(hDevice, IOCTL_CODE, ...) → arbitrary kernel R/W
5
Disable Security & Gain Kernel Access
Use kernel R/W to disable EDR/AV by patching callbacks, removing hooks, or killing protected processes. Full system compromise at Ring 0.
⚠ Result: Kernel-level access, security tools disabled, full system compromise

🛡 Defensive Mitigations

UAC → Always Notify
Set UAC to "Always notify" to prevent silent auto-elevation of trusted binaries
WDAC / Driver Blocklist
Use Microsoft's Vulnerable Driver Blocklist and WDAC policies to prevent known-vulnerable drivers from loading
Monitor Registry Keys
Alert on writes to Shell\Open\Command under HKCU\Software\Classes\ for common hijack targets
HVCI / Secure Boot
Hypervisor-protected Code Integrity prevents unsigned/vulnerable kernel code from executing
Least Privilege
Run users as standard accounts. UAC bypass is irrelevant if the account isn't in the Administrators group
EDR with Kernel Telemetry
Deploy EDR that monitors driver loads, IOCTL patterns, and kernel callback tampering
