ROGUEONE Implant

Component Analysis • Attack Chain Mapping • Capabilities

main.c / command.c
The core execution loop. Initializes C2 network connections and listens for commands. CommandDispatcher maps received TLV payloads to their respective handler functions (e.g., CmdPs, CmdInject).
network.c / protocol.c
Handles TCP socket initialization via Winsock. Implements a custom Type-Length-Value (TLV) protocol for structured C2 communication, sending and receiving dynamic byte arrays or UTF-8 strings.
01
Reconnaissance & Discovery
system.c • process.c • module.c • directory.c
TA0007
.C
System Info (system.c) — Collects hostname, username, and architecture. Uses CheckTokenMembership to determine if the implant is running in an elevated context.
.C
Process Enum (process.c) — Implements both standard API (CreateToolhelp32Snapshot) and undocumented NTAPI (NtQuerySystemInformation) techniques to list running processes, masking the enumeration from basic API hooks.
.C
Module Enum (module.c) — Maps DLLs loaded into a process. The NTAPI version reads the target's Process Environment Block (PEB) directly via NtQueryInformationProcess and traverses the InLoadOrderModuleList.
Attacker Usage
Operators use these files to map the target environment, identify security products, and find suitable target processes (like explorer.exe) for subsequent shellcode injection.
02
Import Address Table (IAT) Parsing
iat.c • Advanced Process Inspection
T1057
.C
Read Remote Memory — Uses NtQueryInformationProcess to locate the remote PEB and ReadProcessMemory to read the remote process's ImageBaseAddress.
.C
Parse PE Headers — Reads the DOS Header and NT Headers from the remote process memory to locate the IMAGE_DIRECTORY_ENTRY_IMPORT.
.C
Extract Functions — Iterates through the IMAGE_IMPORT_DESCRIPTOR structures to list every DLL and function (by name or ordinal) the target process imports.
Attacker Usage
IAT enumeration allows an implant to inspect a running process to see what capabilities it has loaded. This helps identify processes that might have handles to sensitive resources or confirm if a process is hooked by an EDR (by looking for EDR DLLs injected into the IAT).
03
Remote Shellcode Execution
hollower.c • Process Hollowing Technique
T1055.012
.C
Parent PID Spoofing — Finds explorer.exe and uses UpdateProcThreadAttribute to spawn notepad.exe in a suspended state, making it appear as a legitimate child of Explorer.
.C
Locate Entry Point — Queries the suspended process's PEB to find the image base, reads the PE headers, and calculates the exact memory address of the executable's Entry Point.
.C
Write & Execute — Uses WriteProcessMemory to overwrite the legitimate notepad.exe Entry Point with the operator-provided raw shellcode, then calls ResumeThread to execute the malicious payload.
Attacker Usage
Process hollowing allows the implant to run post-exploitation payloads (like Mimikatz, Cobalt Strike beacons, or lateral movement tools) entirely in memory inside a seemingly benign Windows process, evading basic process-level monitoring.
04
Privilege Escalation
uac.c • Auto-Elevation Bypass
T1548.002
.C
Registry Poisoning — Writes the implant's current executable path to HKCU\Software\Classes\Launcher.SystemSettings\Shell\Open\Command.
.C
Trigger Execution — Uses ShellExecuteW to launch slui.exe (a trusted Windows binary).
.C
Bypass Realized — Because slui.exe auto-elevates, it reads the hijacked HKCU registry key and launches the implant as High Integrity (Administrator) without prompting the user.
Attacker Usage
If the initial compromise yields a standard user context, this module is executed to silently escalate privileges to Administrator, unlocking the ability to dump credentials, tamper with the OS, or load kernel drivers.
05
Defense Evasion via BYOVD
driver.c • Bring Your Own Vulnerable Driver
T1562.001
.C
Driver Dropper — Writes a legitimately signed but vulnerable driver (MicrosoftUpdate11.01.sys) to disk.
.C
Kernel Initialization — Assumes the driver was loaded via a service creation (requires Admin, gained in previous step). Opens a handle to \\.\IMFForceDelete123.
.C
Arbitrary Deletion — Sends DeviceIoControl requests (0x8016E000) to the driver, passing paths to AV/EDR files. The driver deletes them from Ring 0, bypassing user-mode file locks and protections.
Attacker Usage
Used as a heavy-handed approach to neutralize endpoint security. By leveraging a signed driver from a vendor (like IObit), Windows allows the kernel code to execute, which the implant then abuses to cripple defensive software blindly.
// Defense & Detection Mitigations
Network Detections: Look for raw TCP connections (non-HTTP/S) communicating via structured 8-byte headers (TLV format), specifically defaulting to port 9001.
Behavioral Analytics: Alert on instances of notepad.exe being spawned with the CREATE_SUSPENDED flag, especially if the parent process is artificially spoofed to be explorer.exe.
Registry & Driver Monitoring: Alert on any modifications to HKCU\Software\Classes\Launcher.SystemSettings\Shell\Open\Command. Utilize Microsoft's Vulnerable Driver Blocklist to stop IMFForceDelete.sys from loading into the kernel.
